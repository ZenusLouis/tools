# Codex Adapter

This folder contains repo-local Codex artifacts generated by GCS Dashboard.

## Layout

- `skills/`: role and workflow skills in Codex `SKILL.md` format.
- `commands/`: command notes that map shared GCS command names to Codex CLI usage.
- `learning/`: Codex-specific lessons and behavior notes.

Shared source-of-truth data lives in `agents/`.

## Settings

GCS-specific Codex runtime settings live in this repo:

- `.codex/settings.json`: shared defaults for the custom GCS Codex adapter.
- `.codex/settings.local.json`: local secret/device overrides, ignored by Git.

Do not put GCS runtime settings in `C:\Users\ACER\.codex\config.toml`; that file belongs to the Codex installation.

## Dashboard logging

The local runner command is defined in `.codex/settings.json`:

```json
{
  "runner": {
    "displayCommand": "powershell -NoProfile -ExecutionPolicy Bypass -File hooks/codex-gcs.ps1 \"<prompt>\""
  }
}
```

Run that command from the repo root when you want sessions to show on the dashboard:

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File hooks/codex-gcs.ps1 "Implement task M1-F1-T1"
```

Or let the settings file drive the command:

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File hooks/run-codex-from-settings.ps1 "Implement task M1-F1-T1"
```

The wrapper auto-loads repo-local settings and dashboard auth from `.codex/settings.json`, `.codex/settings.local.json`, and `apps/dashboard/.env.local`, then sends a bridge heartbeat, runs `codex`, and posts a session log to `/api/log`.

## IDE Chat Sync

Codex IDE chat does not expose the same automatic hook lifecycle as Claude Code. To record the current IDE conversation on the dashboard, log a curated summary:

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File hooks/log-codex-ide-chat.ps1 "Discussed Codex local bridge logging and repo-local settings."
```

The command is also recorded in `.codex/settings.json` under `ideChatSync.displayCommand`.
