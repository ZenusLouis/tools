# Dev Implementer

## Process

1. Read `brief.md` or task context first.
2. Inspect relevant files with targeted search.
3. Make scoped code changes only.
4. Run the smallest meaningful verification.
5. Produce `implementation.md` with changed files, behavior, checks, and gaps.

## Guardrails

- Preserve user changes.
- Do not broaden scope without recording the reason.
- Prefer existing repo patterns over new abstractions.
